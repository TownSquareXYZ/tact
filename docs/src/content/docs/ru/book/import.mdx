Tact позволяет импортировать код Tact и [FunC](https://docs.ton.org/develop/func/overview) — любой файл `.tact` или `.fc`/`.func` можно импортировать в проект с помощью ключевого слова `import{:tact}`.

Кроме того, компилятор Tact имеет универсальный набор стандартных библиотек, которые идут в комплекте, но не включены сразу, см. [Обзор стандартных библиотек](/ref/standard-libraries).

:::caution

  ПРИМЕЧАНИЕ: Весь импортированный код объединяется с вашим, поэтому важно избегать конфликтов имен и всегда дважды проверять исходники!

:::

## Импорт кода Tact

Можно импортировать любой код Tact с помощью оператора `import{:tact}` и указать относительный путь к целевому файлу `.tact`, например:

```tact
import "./relative/path/to/the/target/tact/file.tact";
```

Также возможно указать родительские каталоги (`../`):

```tact
import "../subfolder/imported/file.tact";
```

## Импорт кода FunC

Можно импортировать код, написанный на FunC, напрямую, как это делается с помощью импорта кода Tact:

```tact
// Относительный импорт
import "./relative/path/to/the/target/func/file.fc";

// Указание родительских каталогов
import "../subfolder/imported/func/file.fc";
```

Но чтобы использовать функции из такого файла, их сначала нужно объявить как `native`. Например, когда стандартная библиотека [`@stdlib/dns`](/ref/stdlib-dns) использует файл FunC `dns.fc`, она сопоставляет функции FunC с функциями Tact следующим образом:

```tact
// Код FunC, расположенный в файле прямо рядом с текущим кодом Tact:
import "./dns.fc";

// Преобразование подписей функций из FunC в Tact:
@name(dns_string_to_internal)
native dnsStringToInternal(str: String): Slice?;
```
