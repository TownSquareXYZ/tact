ABI определяет, как взаимодействовать со смарт-контрактами. Он содержит информацию о получателях контракта, структурах данных и т. д.

## Мотивация

ABI — это важный инструмент, который позволяет разработчикам создавать удобные привязки, пользовательские интерфейсы и т. д. Одним из лучших вариантов использования для потребителей будет использование DAO и возможность подтверждения того, что именно он пытается сделать перед подписанием транзакции.

## Руководство

Этот OTP основан на типах, определенных в TLB+, и рекомендуется ознакомиться с ними перед чтением этого OTP.

## Спецификация

ABI - это JSON-файл:

```json
{
  "name": "MyContract",
  "types": [
    {
      "name": "MyRequest",
      "header": 12315123,
      "fields": [
        {
          "name": "queryId",
          "type": {
            "kind": "simple",
            "type": "int",
            "optional": false,
            "format": "uint256"
          }
        }
      ]
    }
  ],
  "receivers": [
    { "type": "binary", "kind": "internal", "name": "MyRequest" },
    { "type": "binary", "kind": "internal" },
    { "type": "comment", "kind": "internal", "comment": "Vote!" },
    { "type": "comment", "kind": "internal" },
    { "type": "empty", "kind": "internal" }
  ],
  "getters": [
    { "name": "getOwner", "type": "address", "args": [] },
    {
      "name": "getBalance",
      "type": "coins",
      "args": [
        {
          "name": "invested",
          "type": {
            "kind": "simple",
            "type": "uint",
            "format": "coins"
          }
        }
      ]
    }
  ],
  "errors": {
    "123": "Описание ошибки",
    "124": "Деление на ноль"
  }
}
```

## Недостатки

- Двоичное и компактное представление ABI могло бы быть лучше, но сейчас это не является критическим.
