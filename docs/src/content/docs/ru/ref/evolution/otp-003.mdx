Это предложение определяет, как сообщать об ABI контракта, используя ссылку IPFS.

## Мотивация

Обычно ABI поставляется отдельно с использованием стороннего сервиса или через какой-либо репозиторий на GitHub. Это предложение предполагает добавление нового самостоятельного отчета об ABI контракта с помощью ссылки на IPFS. Это позволит нам избежать любой зависимости от третьих лиц и позволит любому создавать инструменты, которые полагаются на ABI, такие как проводники, кошельки и т. д.

## Спецификация

Чтобы поддержать это предложение, контракт должен реализовать OTP-001 и сообщать интерфейс `org.ton.abi.ipfs.v0`. Затем необходимо реализовать get-метод `get_abi_ipfs`, который возвращает строку со ссылкой на файл ABI в IPFS. Ссылка должна быть в формате `ipfs://<hash>`.

## Недостатки

- Невозможно обновить ABI без обновления контракта. Этот недостаток существует только для жестко закодированных ссылок.
