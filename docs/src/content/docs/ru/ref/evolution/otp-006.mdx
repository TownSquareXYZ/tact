Это предложение определяет способ упаковки компилируемых контрактов, их зависимостей и всех связанных метаданных в один файл.

## Мотивация

Общий формат пакета необходим для упрощения процесса развертывания и обновления контрактов с использованием различных инструментов без необходимости их настройки.

## Спецификация

Файл пакета имеет расширение `.pkg` и является файлом JSON:

```json
{
  "name": "My Contract",
  "code": "... boc кода ...",
  "abi": "Строка ABI для загрузки в IPFS или Ton Storage",
  "init": {
    "kind": "direct", // Означает, что этот контракт может быть развернут как есть
    "args": {
      // ... Аргументы в формате ABI
    },
    "prefix": {
      // Необязательный префикс для состояния инициализации контракта
      "bits": 0, // Количество бит в префиксе
      "value": 0 // Значение префикса
    },
    "deployment": {
      "kind": "system-cell", // Означает, что этот контракт может быть развернут как есть
      "system": "... boc системной ячейки ..."
    }
  },
  "sources": {
    "file.ton": "... исходный файл в кодировке base64 ..."
  },
  "compiler": {
    "name": "func",
    "version": "0.4.1",
    "parameters": "..." // Необязательные строковые параметры
  }
}
```
